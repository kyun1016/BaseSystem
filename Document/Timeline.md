# 육성 시뮬레이션 게임 개발 TODO List

## Phase 1: 데이터 아키텍처 및 파이프라인 (Data Pipeline)
*기획 데이터의 무결성을 보장하고, 클라이언트에서 이를 효율적으로 끌어다 쓰기 위한 기반 작업입니다.*

- [x] **아이템 데이터 파이프라인 구축**
  - [x] CSV 기반 데이터 테이블 스키마 설계 (`ID, Name, Effects, Conditions` 등)
  - [x] 리플렉션(Reflection)을 활용한 CSV -> ScriptableObject 자동 파싱 엔진 구현
  - [x] 특수 리스트 자료형(`ItemEffect`, `ItemCondition`) 커스텀 파싱 로직 구현
  - [x] CSV에 존재하지 않는 더미 에셋(Ghost Data) 자동 추적 및 삭제 로직 추가
- [ ] **데이터 파서 확장**
  - [ ] 교육/아르바이트 스케줄 CSV 파싱 로직 추가 (`GameDataParser` 분기 연동)
  - [ ] 이벤트 및 조건(Event & Flag) 데이터 파싱 로직 추가
- [ ] **리소스 관리 (옵션)**
  - [ ] Addressables 시스템 세팅 및 SO 내의 `SpritePath`로 에셋 비동기 로드 연결

## Phase 2: 핵심 육성 시스템 (Core Raising Systems)
*게임의 메인 루프인 시간의 흐름과 스탯 변화를 다루는 코어 로직입니다.*

- [ ] **스탯 시스템 (Stat System)**
  - [ ] 스탯 컨테이너 구현 (각종 스탯을 `Dictionary`나 열거형 배열로 관리)
  - [ ] 영구 스탯(Base Stat)과 임시 스탯(Modifier - 장비/버프) 분리 계산 로직
  - [ ] 최대/최소치 제한(Clamp) 및 변화량 검증 로직
- [ ] **시간 및 스케줄 시스템 (Time & Schedule System)**
  - [ ] 달력(시간 흐름) 관리 매니저 구현 (일/주/월/년 단위 전환)
  - [ ] 플레이어가 입력한 스케줄 리스트를 큐(Queue) 구조로 저장
  - [ ] 큐에 담긴 스케줄을 순차적으로 꺼내며 실행(성공/실패/대성공 연산)하는 프로세서 구현
- [ ] **아이템 시스템 실제 연동**
  - [ ] 인벤토리 시스템 구현 (아이템 획득, 소비, 장착 로직)
  - [ ] 아이템 사용 시 파싱된 `Effects` 리스트를 스탯 시스템에 반영하는 로직

## Phase 3: 상호작용 및 이벤트 (Events & Interaction)
*육성 시나리오의 다양한 분기와 스토리를 제어하는 시스템입니다.*

- [ ] **플래그 및 조건 시스템 (Flag & Condition System)**
  - [ ] 게임 내 모든 선택과 달성 여부를 기록하는 글로벌 플래그 저장소 (`Dictionary<string, int>`)
  - [ ] SO에 정의된 조건(`ItemCondition` 등)을 실제 게임 상태와 비교하여 참/거짓을 반환하는 검사기(Evaluator) 구현
- [ ] **대화 및 이벤트 (Dialogue System)**
  - [ ] 텍스트 어드벤처 툴(Yarn Spinner, Ink 등) 도입 검토 및 유니티 연동
  - [ ] 특정 시간, 특정 스탯 조건을 만족했을 때 자동으로 이벤트를 발생시키는 트리거 시스템 구현

## Phase 4: UI 바인딩 및 세이브/로드 (UI & State Management)
*플레이어에게 정보를 시각화하고, 게임의 진행 상태를 안전하게 저장합니다.*

- [ ] **반응형 UI (Reactive UI)**
  - [ ] 스탯 데이터가 변경될 때마다 이벤트를 발행(Publish)하는 구조 설계
  - [ ] UI 스크립트가 로직을 직접 참조하지 않고 이벤트를 구독(Subscribe)하여 게이지/텍스트를 갱신하도록 분리
- [ ] **상태 중앙화 및 직렬화 (Save & Load)**
  - [ ] 현재 턴, 모든 스탯, 소지 아이템, 플래그 등 '저장해야 할 모든 데이터'를 순수 C# 클래스(`GameState`) 하나로 캡슐화
  - [ ] `GameState` 객체를 JSON 포맷으로 직렬화하여 로컬 기기에 저장
  - [ ] JSON 파일을 역직렬화하여 게임을 완벽하게 이전 상태로 복구하는 로드 매니저 구현
